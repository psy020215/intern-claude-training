01 · 개요

개요

이 문서는 SBML Lab 신규 인턴이 실제 생물정보학 워크플로우 안에서 Claude Code — AI 코딩 어시스턴트 — 를 활용하는 방법을 배우기 위한 6주 훈련 커리큘럼을 기술합니다. 커리큘럼은 ChIP-exo 연구의 전체 분석 파이프라인을 다룹니다: 레퍼런스 데이터 수집, 리드 정렬, 모티프 발견, 생물학적 해석. 마지막 2주는 공개된 FUR 데이터를 이용한 독립 미니 프로젝트로 진행됩니다.

커리큘럼은 GitHub Codespaces에서 실행되는 Jupyter 노트북으로 제공되어 로컬 소프트웨어 설치가 전혀 필요하지 않습니다. 랩에서 사용하는 모든 생물정보학 도구 — bowtie2, samtools, MEME Suite, Biopython — 는 클라우드 환경에 사전 설치되어 있습니다. 인턴은 Codespace 실행 후 개인 Claude Pro 계정으로 Claude Code를 인증합니다.

핵심 요약

  • 6주에 걸친 5개 모듈이 랩의 실제 연구에서 사용하는 ChIP-exo 분석 파이프라인 전체를 다룹니다.
  • 로컬 설치 불필요: GitHub Codespaces가 모든 필요 도구를 포함한 conda 환경을 사전 구성합니다.
  • 마지막 2주는 Seo et al. 2014 FUR ChIP-exo 데이터셋을 이용한 독립 미니 프로젝트로, 새로운 데이터 없이 독창적인 생물학적 질문에 답합니다.
  • 모듈 1–4는 비공식 평가입니다. /log 세션 기록으로 진도를 확인하며, 공식 100점 루브릭은 미니 프로젝트에만 적용됩니다.

02 · 커리큘럼 구성

커리큘럼 구성

커리큘럼은 6주에 걸쳐 5개 모듈로 진행됩니다. 모듈 1–4는 각각 독립적인 Jupyter 노트북이며, 마지막 2주는 독립 미니 프로젝트에 할당됩니다.

모듈 일정

주차모듈주제
1주Module 1Claude Code 입문
2주Module 2Claude Code를 활용한 파일 파싱 및 데이터 처리
3주Module 3Claude Code를 활용한 NGS 정렬 파이프라인
4주Module 4문헌 통합, Biopython, 모티프 분석
5–6주Module 5독립 미니 프로젝트

5개 노트북은 모두 notebooks/ 디렉토리에 있습니다. 레퍼런스 데이터와 다운로드 파일은 data/reference/에 중앙 관리됩니다. 배포 전 지도 교수가 사전 제공하는 유일한 파일은 랩의 내부 E. coli K-12 MG1655 주석 GFF입니다. 그 외 모든 데이터는 인턴이 실습의 일환으로 직접 수집합니다.

랩 제공 스킬

.claude/skills/에 Markdown 스킬 파일로 정의된 세 가지 슬래시 명령어가 첫 세션부터 사용 가능합니다.

스킬설명
/log 매 세션 종료 시 실행하여 완료 사항, 발생한 오류, 학습 내용을 기록합니다. 6주 전체에 걸친 랩 노트북을 누적 작성합니다.
/debug 명령어나 스크립트가 실패할 때 실행하는 구조화된 디버깅 프로토콜입니다 (재현 → 위치 특정 → 수정 → 검증 → 문서화). 지도 교수에게 질문하기 전에 먼저 사용해야 합니다.
/explain 도구, 알고리즘, 개념을 랩의 문맥 — 생물종, 파일 형식, 실험 기법 — 에 맞춰 설명합니다. 외부 검색 전에 먼저 사용합니다.
모든 코드 셀은 빈 상태로 시작합니다. 어떤 실습에도 정답이 미리 제공되지 않습니다. 데이터 수집 명령어조차 주어지지 않으며, 인턴은 Claude Code를 통해 무엇을 실행할지 스스로 파악하고 공개 데이터베이스를 직접 탐색합니다.
03 · 모듈 상세 설명

모듈 상세 설명

모듈 1–4는 순차적으로 심화됩니다. 새로운 도구를 도입하기 전에 개념적 토대를 먼저 확립합니다. 생물학적 맥락은 연속적으로 유지됩니다: 동일한 생물종, 주석 파일, 데이터 타입이 모든 모듈에 걸쳐 사용됩니다.

1주차 Module 1 — Claude Code 입문

학습 목표: Claude Code가 무엇이고 챗봇과 어떻게 다른지 이해하기; 효과적인 프롬프트 작성법 익히기; Claude의 출력을 비판적으로 검토하는 습관 기르기; 제공된 세 가지 랩 스킬 사용하기; Claude Code의 도움을 받아 첫 커밋 만들기.

Module 1은 이후 모든 모듈의 개념적 기반을 구축합니다. 인턴은 먼저 Claude Code의 작동 방식을 배웁니다: 터미널에서 실행되고, 파일을 읽고 쓰며, 명령어를 실행합니다 — 웹 인터페이스가 아닙니다. 개념 섹션에서는 세 가지 사용 모드(코드 작성, 코드 설명, 코드 디버깅)와 입력의 구체성이 출력의 품질을 결정한다는 원칙을 설명합니다.

프롬프팅 섹션은 비판적 검증에 관한 명시적인 안내로 마무리됩니다: Claude는 틀릴 수 있습니다 — 플래그가 약간 잘못되거나, 매뉴얼 페이지와 일치하지 않는 매개변수를 확신 있게 제시하거나, 문헌과 맞지 않는 생물학적 주장을 할 수 있습니다. 인턴은 결과가 의외이거나 논문에서 읽은 내용과 맞지 않을 때 확인하는 본능을 개발하도록 권장됩니다. 이 습관은 여기서 도입되어 이후 모듈 전체에서 개념 체크를 통해 강화됩니다.

실습 과제

  1. 자연어로 Python 스크립트 작성. 인턴은 별 삼각형 패턴을 평문으로 설명하고 Claude Code에게 스크립트 생성을 요청합니다. 코드 작성 모드와 후속 프롬프트를 통한 반복 개선이 목표입니다.
  2. 버그 있는 스크립트 디버깅. 제공된 카운트다운 스크립트에는 의도적인 버그가 있습니다 — 잘못된 range() 인수가 예외 없이 잘못된 출력을 만듭니다. 인턴은 /debug를 사용해 근본 원인을 찾고 최소한의 수정을 적용합니다.
  3. 세 가지 스킬 읽기 및 실행. 인턴은 세 개의 스킬 파일을 읽고, 각각 최소 한 번씩 실행하며, 작동 방식과 사용 시점에 관한 질문에 답합니다.
  4. 랩 컨텍스트로 스킬 확장. 인턴은 lab-context.md에서 구체적인 세부 정보를 하나 골라 /explain 스킬에 추가합니다. 스킬이 단순한 Markdown 파일임을 — 읽고, 수정하고, 확장할 수 있는 — 그리고 특정 컨텍스트에 맞게 커스터마이징하면 출력이 향상된다는 것을 배웁니다.

/log로 작성한 세션 기록이 1주차 제출물입니다.

2주차 Module 2 — 파일 파싱 및 데이터 처리

학습 목표: Claude Code를 사용해 자연어 설명으로 GFF 파싱 스크립트 생성 및 개선하기; pandas 오류 진단 및 수정하기; pandas의 로딩 및 인덱싱 모델 이해하기.

Module 2는 GFF 파일 형식 — 게놈 특징의 주요 데이터 표현 방식 — 과 구조화된 데이터 처리를 위한 pandas 라이브러리를 소개합니다. 개념 섹션에서는 GFF 컬럼 구조(9개 탭 구분 컬럼, 1-based 좌표, attribute 필드 문법)와 이것이 pandas DataFrame에 어떻게 매핑되는지 설명합니다.

모든 실습은 랩의 내부 E. coli K-12 MG1655 주석 파일 (ec_annotation_20100903_DHK_cSRNA_with_ortho.gff)을 사용합니다. 이 파일은 랩의 발표 연구에서 도출된 TSS 위치, sRNA 주석, 유전자 특징을 포함하며, Module 4와 미니 프로젝트에서도 동일하게 사용됩니다.

실습 과제

  1. 자연어 → GFF 파싱 스크립트. 인턴은 필터링 작업을 평문으로 설명하고 Claude Code에게 스크립트 생성을 요청합니다. 반복 협업을 배우기 위해 최소 두 번의 후속 개선 프롬프트가 필요합니다.
  2. 버그 있는 pandas 스크립트 디버깅. 제공된 스크립트에는 두 가지 버그가 있습니다: header=None 없이 GFF 파일을 로드해 첫 번째 데이터 행이 컬럼명으로 잘못 해석되고, 컬럼 접근이 정수 인덱스 대신 문자열 이름을 사용합니다. 인턴은 /debug를 사용해 KeyError를 재현하고, 두 가지 근본 원인을 파악하여 수정합니다.
  3. iterrows()를 언제 사용하나. 인턴은 Claude Code에게 iterrows()가 적합한 경우와 벡터화 연산이 더 나은 경우를 설명해 달라고 요청합니다. 개념 체크는 Claude Code에게 확인하기 전에 한 문장 규칙을 먼저 작성하도록 합니다.

3주차 Module 3 — NGS 정렬 파이프라인

학습 목표: bowtie2 정렬 알고리즘과 samtools 후처리 단계를 개념적으로 이해하기; plan mode를 사용해 실행 전에 전체 파이프라인 생성 및 검토하기; pysam을 사용해 커스텀 BAM-to-GFF 변환 스크립트 작성하기.

Module 3은 기술적으로 가장 실질적인 모듈입니다. 랩에서 전사 인자 결합 부위 매핑에 사용하는 단일 말단 ChIP-exo 정렬 전체 워크플로우를 다룹니다: 레퍼런스 인덱싱, bowtie2로 리드 정렬, samtools로 좌표 정렬된 BAM 생성, MetaScope 시각화를 위한 GFF 형식 변환. 파일 형식 참조 표 (FASTQ, FASTA, SAM, BAM, GFF)를 통해 명령어를 실행하기 전에 실제 데이터 흐름을 파악합니다.

개념 섹션에서는 bowtie2의 FM-index와 seed-and-extend 모델, 그리고 samtools의 좌표 정렬 요구 사항을 — 사전 작성된 설명 없이 — 모두 Claude Code를 통한 직접 프롬프트로 설명합니다.

실습 과제

  1. bowtie2 이해. 어떤 명령어도 실행하기 전에, 인턴은 Claude Code를 사용해 bowtie2가 정렬을 찾는 방법과 주요 플래그가 무엇을 제어하는지 이해합니다. 인턴 자신의 언어로 3문장 요약이 필요합니다.
  2. 정렬 전 인덱싱 이유 추론. Claude Code에게 먼저 묻지 않고, 인턴은 BAM 파일을 인덱싱 전에 정렬해야 하는 이유를 직접 설명한 후 Claude Code로 검증합니다.
  3. Plan mode 파이프라인. Plan mode(Shift+Tab, Pro 계획 필요)를 사용해 인턴은 레퍼런스 인덱싱부터 정렬된 BAM까지 전체 파이프라인을 생성합니다. 모든 제안 단계를 검토하고 최소 한 가지 수정 사항을 파악한 후 승인해야 합니다. Plan mode의 첫 공식 사용입니다.
  4. ChIP-exo 리드 탐색 및 다운로드. 인턴은 GEO series GSE54901에 직접 접속해 적절한 단일 말단 ChIP-exo SRR accession을 스스로 찾고 Claude Code를 사용해 다운로드합니다. 명령어는 미리 제공되지 않습니다.
  5. SAM FLAG 값 해독. 인턴은 이전에 찾아본 적 없는 FLAG 값을 선택해 Claude Code를 사용해 비트 단위 구성 요소를 해독합니다. SAM 명세와 이진 인코딩을 소개합니다.
  6. makegff.py 작성. 인턴은 pysam을 사용해 랩의 커스텀 BAM-to-GFF 변환 스크립트를 작성합니다. 스크립트는 정렬되고 인덱싱된 BAM 파일을 읽고 1-based 좌표와 가닥 정보를 포함해 정렬된 리드 당 하나의 GFF 줄을 출력합니다: 
    NC_000913.3  makegff  read  <start>  <end>  1  <strand>  .  name=<read_name>
    이 실습은 BAM 형식, GFF 컬럼 구조, pysam API, 좌표 규칙 등 앞서 배운 모든 지식을 하나의 결과물로 통합합니다.

레퍼런스 게놈(NC_000913.3)도 NCBI accession만 힌트로 제공된 상태에서 인턴이 Claude Code를 사용해 직접 수집합니다.

4주차 Module 4 — 문헌 통합, Biopython, 모티프 분석

학습 목표: Claude Code를 사용해 발표 논문에서 구조화된 정보 추출하기; GEO에서 공개 ChIP-exo 데이터 수집하기; 서열 추출을 위한 Biopython 스크립트 작성하기; MEME Suite로 모티프 발견 실행하기; 모티프 결과를 생물학적으로 해석하기.

Module 4는 앞서 배운 모든 기술을 실제 공개 데이터셋과 연결합니다: Seo et al. (2014)의 E. coli FUR(Ferric Uptake Regulator) ChIP-exo 연구로, GEO accession GSE54901에서 이용 가능합니다. FUR은 E. coli에서 철분에 반응하는 전역 전사 인자로, 예상 모티프(Fur box)가 문헌에서 알려져 있어 훈련 사례로 적합합니다.

개념 섹션에서는 MEME 알고리즘(서열 윈도우에 대한 기댓값-최대화)과 E-value 해석을 소개합니다. 각 피크가 정규 결합 모티프를 포함할 수도, 포함하지 않을 수도 있는 ChIP-exo 데이터에 적합한 -mod zoops 모델을 사용합니다. 인턴은 Claude Code에게 확인하기 전에 FUR이 이중 가닥 DNA에 결합한다는 사실을 고려하는 플래그가 무엇인지 먼저 추론합니다.

실습 과제

  1. Seo et al. 2014 읽기. 인턴은 논문을 읽고 3문장 요약을 작성합니다: 실험이 측정한 것, 사용된 생물종과 조건, 주요 발견. 개념 셀에는 결과가 미리 기술되어 있지 않습니다.
  2. FUR regulon 교차 확인. Claude Code를 사용해 FUR regulon에 대한 다른 공개 레퍼런스 유전자 세트와 그 결정 방법을 파악합니다.
  3. MEME 이해. 어떤 명령어도 실행하기 전에, 인턴은 MEME의 작동 원리와 E-value의 의미를 3문장으로 설명한 후 Claude Code로 검증합니다.
  4. Claude Code로 GEO에서 데이터 다운로드. 인턴은 GEO series URL을 Claude Code에 제공하고 FUR ChIP-exo 피크 GFF 파일과 레퍼런스 게놈 FASTA를 다운로드하도록 요청합니다.
  5. Biopython 서열 추출 스크립트 작성. 다운로드한 피크 좌표와 레퍼런스 게놈을 사용해, 각 피크 summit 주변 ±50 bp 윈도우를 추출하고 MEME 입력용 multi-FASTA 파일로 저장하는 Biopython 스크립트를 작성합니다.
  6. MEME 실행 및 결과 해석. Claude Code의 도움으로 MEME 명령어를 구성하고 추출된 서열에 대해 실행합니다. 그런 다음 상위 모티프가 생물학적으로 의미 있는지 근거와 함께 평가합니다.
  7. TSS 거리 분석. 랩의 주석 GFF를 사용해, 각 FUR 결합 부위와 가장 가까운 주석된 TSS 사이의 거리를 계산하는 Python 스크립트를 작성합니다.
04 · 독립 미니 프로젝트

독립 미니 프로젝트

미니 프로젝트는 Module 4에서 사용한 동일한 FUR ChIP-exo 데이터셋에 대한 개방형 독립 분석입니다. 새로운 데이터셋이 도입되지 않습니다. 인턴은 Codespace에 설치된 모든 도구를 사용할 수 있으며, 질문에 추가 패키지나 데이터가 필요한 경우 Claude Code를 사용해 수집할 수 있습니다.

학습 목표: 실제 데이터셋에서 독창적인 생물학적 질문 수립하기; Claude Code의 도움을 받아 전체 분석 파이프라인 설계 및 실행하기; 결과를 발표 문헌의 맥락에서 해석하기; 재현 가능한 분석 보고서 작성하기.

프로젝트 구성

Part 1 — 문헌 맥락

인턴은 Seo et al. 2014 이외의 FUR에 관해 알려진 내용을 조사합니다: 다른 ChIP 연구, 알려진 Fur box 컨센서스, FUR regulon의 유전자들, 조건 의존적 결합 행동.

Part 2 — 생물학적 질문

인턴은 FUR ChIP-exo 데이터에서 답할 수 있는 구체적이고 생물학적으로 동기 부여된 질문 하나를 제안합니다. 허용 가능한 질문 예시: "높은 피크 점수를 가진 FUR 결합 부위는 더 강한 모티프 일치를 보이는가?", "FUR 결합 부위는 단백질 코딩 유전자에 비해 sRNA 유전자 주변에 농축되어 있는가?", "FUR 부위의 GC 함량은 무작위 게놈 윈도우와 어떻게 비교되는가?" 모호한 질문은 허용되지 않으며, 분석 진행 전 지도 교수 체크인이 필요합니다.

Part 3 — 분석 계획

인턴은 독립적으로 평문 계획을 작성합니다. 이후 plan mode를 사용해 Claude Code에게 전체 파이프라인 초안 작성을 요청하며, 실행 승인 전에 각 제안 단계에 주석을 달아야 합니다.

Part 4–6 — 실행, 결과, 보고서

인턴은 Claude Code의 도움을 받아 분석을 실행합니다. 모든 오류는 /debug로 처리하고, 모든 세션은 /log로 기록합니다. 최소 하나의 그림을 생성해야 합니다. 노트북의 Markdown 셀로 작성하는 최종 보고서는 생물학적 질문과 동기, 방법, 그림을 포함한 결과, 해석을 담습니다. Claude Code는 글쓰기 보조에 활용할 수 있으나 필수는 아닙니다.

설계 원칙

1 · 인턴은 사전 작성된 셀이 아닌 Claude Code를 통해 답을 발견합니다

어떤 실습에도 정답이 미리 제공되지 않습니다. 모든 코드 셀은 빈 상태로 시작합니다. 데이터 수집 명령어조차 — SRA accession 탐색, 레퍼런스 게놈 다운로드 — 주어지지 않습니다. 인턴은 Claude Code를 사용해 무엇을 실행할지 파악하고 공개 데이터베이스를 직접 탐색합니다.

2 · 모든 모듈은 실제 랩 데이터와 실제 도구를 사용합니다

Module 2는 실제 랩 주석 GFF를 사용합니다. Module 3은 NCBI SRA의 실제 ChIP-exo FASTQ를 사용합니다. Module 4는 GEO의 발표된 Seo et al. 2014 FUR 데이터셋을 사용합니다. 미니 프로젝트는 동일한 데이터에 대한 독창적 분석입니다. 인턴은 훈련을 마칠 때 실제 랩 연구에서 접하게 될 동일한 도구로 동일한 데이터 타입을 처리해 본 경험을 갖게 됩니다.

3 · Claude Code는 이해를 위한 도구이지 이해를 피하는 지름길이 아닙니다

각 모듈에는 인턴이 Claude Code에게 묻기 전에 자신의 설명을 먼저 작성해야 하는 개념 체크가 최소 하나 포함됩니다. Plan mode는 모든 파이프라인 생성 프롬프트에서 필수이며, 각 제안 단계의 검토와 승인을 강제합니다. Claude Code 활용에 대한 평가 기준은 코드를 생성하는 능력이 아닌 생성된 코드를 설명하는 능력을 명시적으로 평가합니다.

4 · 비판적 검증은 단계가 아닌 습관입니다

인턴은 Module 1에서 Claude가 틀릴 수 있다는 사실을 배웁니다 — 플래그가 약간 잘못되거나, 논문과 맞지 않는 생물학적 주장을 할 수 있습니다. 형식적인 교차 확인 단계를 강제하는 대신, 커리큘럼은 이것을 본능으로 키웁니다: 개념 체크는 양방향으로 진행되며(묻기 전에 먼저 쓰고, 받은 후에 검증), 인턴은 결과나 설명이 올바르지 않다고 느껴질 때 이를 알아채고 직접 조사하는 것을 기대합니다.

Codespace 구성을 인턴에게 배포하기 전에 다음 단계를 완료합니다:

  3. Codespace를 테스트로 실행하여 pysam을 포함한 모든 도구가 정상 작동하는지 확인합니다.
